# 命运线设计
- 命运线用于控制玩家的一个长线目标

# 命运线结构设计
- 命运有多个阶段
- 每个阶段有触发条件，可以是多少天后，遇到某个人后，或者获取某个物品或者一个encounter，或者完成某个任务
- 每个阶段有必然触发一个命运quest

# 命运,quest和encounter的关系
- 命运是一系列命运quest的集合
- 而quest是一系列目标的集合
- encounter只是遭遇，可以用于启动quest或者作为quest的一部分

# 命运线的触发
- 先不考虑触发，只在debug界面加一个手动生成命运线和查看命运信息的按钮。

# 命运线的结构
- 命运事件描述
- 命运事件触发条件
- 命运事件是否已经触发（存在存档里面）

# AI工具
- 生成命运线

- 获取命运线相关信息
- 检查命运事件是否触发

- 本质上也是quest，应该遵从quest的设计
