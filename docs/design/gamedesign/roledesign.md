# 角色框架与玩家数据设计

- 使用5E规则打底

## 目标

- 定义统一角色框架，覆盖玩家角色、NPC 角色、怪物角色。
- 提供可直接落地到数据模型的 DND 5E 角色卡模板。
- 将玩家信息与地图/聊天流程解耦，支持长期成长扩展。
- 保证后续扩展（装备、任务、战斗）时不破坏已有存档。

## 角色类型

- `player`：玩家角色，由玩家创建与长期成长。
- `npc`：非玩家角色，由系统生成或剧情配置。
- `monster`：怪物角色，用于遭遇与战斗。

## 玩家数据面板规则

### 核心体验规则

- 玩家数据面板由 Debug 面板进入。
- 修改后立即生效，不需要重启会话。

### 交互约束

- 必填字段不可为空（如角色名、核心标识）。
- 非法数值输入应被阻断或回退到安全默认值。
- 保存成功后应刷新当前会话中依赖该数据的功能。

## BUFF属性枚举

- BUFF名称
- BUFF描述
- BUFF持续时间

## 属性计算公式

- AC = 武器攻击力 + 力量加值（力量武器）+ 敏捷加值（敏捷武器）+ BUFF影响
- DC = 敏捷加值（若没有穿重甲） + BUFF影响
- 所有当前值 = 基础值 + BUFF影响

## NPC单聊体验规则

- 玩家点击“可交互NPC”按钮后，弹出行为输入框，同聊天框输入一样，让玩家描述首次和该NPC交互执行的动作和语言，可以只输入单个即可以只有动作或语言。
- 玩家执行动作需要进行检定。
- 然后AI根据玩家的动作和NPC的设定去和检定结果做出反馈。
- 玩家对NPC提出的所有要求都要进行检定，根据检定结果和NPC设定进行反馈。
- 玩家主动返回主聊天即为离开该NPC，玩家可以通过按钮离开，也可以通过和NPC单聊时描述的动作进行离开，所以需要准备AI工具
- 聊天记录必须结构化，同时AI需要返回执行的动作反应（表情，小动作等），和语言，NPC可以只有动作没有语言回复
- 和NPC单聊会有NPC健谈值，作为NPC是否还愿意继续回复的标杆，同时这个值会受到，玩家首次交互，NPC设定性格，以及谈话内容影响，每次聊天都会对该值进行修正，总体是不断消耗该值的，但说一些NPC感兴趣的东西会恢复一点，当这个值为0时，NPC不再理会玩家。
- 健谈值会根据时间恢复，在玩家和NPC进入单聊时，根据和上一次单聊的时间差回复健谈值。

- 我更想要NPC自己对当前局势进行思考做出动作的表现，所以更改一下区域群聊规则
- 采用轮流回复的规则，玩家先采取动作和语言，检定后GM先给出玩家的动作带来的最直接的反馈，然后再请求NPC根据当前局势和现在扮演的角色，尝试去执行动作和语言，然后经过检定后，GM给出反馈，然后来到可互动NPC，可互动NPC也是自行决策行动，GM检定给反馈，就像飞行棋一样轮流进行动作并获得反馈。
- 如果在遭遇中，任何人的行为都会导致局势值变化。

## 区块声望

- 玩家在每个区块都有声望值，帮助区块的人，遵守区块的秩序便能保持或者提升声望
- 越做怪事情，或者违反区块的规则，则会降低在该区块的声望
- 区块声望越高，NPC越容易提升好感
- 区块声望越低，NPC越容易降低好感
- NPC的行为算作队伍行为，也会对区块声望造成影响


## NPC和队友的欲望

- 为了让角色扮演更加沉浸式，队友或者可互动NPC会有想要的东西，或者想做的事情
- 可互动NPC可以通过给玩家发布quest来请求玩家寻找或者给予东西
- 队友则会在非遭遇时闲置时例如旅行到一个新的地方会想要一些东西，当然也要根据性格或者自身的秘密来决定，提出一些需求，为游戏提供动力。
- 我希望角色扮演能有一个基础的prompt:角色会有自己的想法，需要沉浸式根据角色的性格身世秘密等来演绎角色


## 队友的故事

- 当玩家和队友旅行一段时间后，且好感度达到时，闲置时，会提出和玩家聊一下关于自己的事情


## 背包设计

- 当前金币数量
- 其他物品列表
- 所有物品都是Item包括玩家的武器、护甲等
- 玩家在背包中可以装备武器和护甲和选取道具
- 武器和护甲类型在背包中点击后可以选择是否装备
- 其他类型的Item点击，弹出聊天框，玩家可以输入如何使用或者观察这个Item,输入prompt；AI 返回是否需要检定和时间消耗
